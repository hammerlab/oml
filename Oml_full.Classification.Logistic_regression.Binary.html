Functor Oml_full.Classification.Logistic_regression.Binary

module Binary: 
functor (D : Oml.Classification.Input_interfaces.Continuous_encoded_data) -> sig .. end
Use Logistic Regression to estimate log-odds for each of the quantitative features in the encoded data, per class.

Unless specified via the classes argument to estimate, the first class encountered will be considered the base_class. All other classes will be considered as not base_class and eval will reflect that, by returning the same probability. For multi-class logistic regression see Multiclass.

A constant 1 is added to all encoded features by estimate, there is no need to add one with encoding.

val opt : ?lambda:float -> ?tolerance:float -> unit -> opt
opt ~lambda ~tolerance () a constructor for the optional arguments,
lambda : The regularization parameter of the fitting cost. The default value is 0.0001.
tolerance : The accuracy tolerance of the underlying L-BFGS convergence algorithm. Convergence stops once successive evaluations of cost function (scaled) are less than tolerance * epsilon_float. 1e4 is used by default. See the method for details.
val coefficients : t -> float array
coefficients t the weights assigned to the log-odds of the features.

The first coefficient is for a constant term and therefore there will always be 1 more than the size of features.

val base_class : t -> class_
base_class t returns the class C against which the log-odds are computed (and hence coefficients).
